## Recording Script (2-3 minutes)

### Introduction (15 seconds)
```
"Hello! This is my Weather Voice Assistant project for Vaiu AI.
It's built with LiveKit, Python, and OpenWeatherMap API.
Let me show you how it works."
```

### Demonstration (2 minutes)

#### 1. Voice Query - Current Weather (30 seconds)
**Action**: Click "Start Voice" button

**Say**: "What's the weather in Mumbai?"

**Wait for**: 
- Voice recognition to capture
- API call to complete
- Voice response to play

**Explain**: "As you can see, it recognized my voice, fetched the weather data, and responded naturally."

#### 2. Text Query - Another City (20 seconds)
**Action**: Type in text box: "Weather in Bangalore"

**Click**: Send button

**Explain**: "You can also type your queries. It works the same way."

#### 3. Forecast Query (30 seconds)
**Action**: Use voice or text

**Say/Type**: "Will it rain in Pune tomorrow?"

**Explain**: "It can also provide weather forecasts and rain probability."

#### 4. Multiple Queries (20 seconds)
**Action**: Ask about another city quickly

**Say/Type**: "How about Delhi?"

**Explain**: "You can ask about multiple cities in succession."

#### 5. Error Handling (20 seconds)
**Action**: Use voice or text

**Say/Type**: "Weather in XYZ123"

**Wait for**: Error message

**Explain**: "Notice how it gracefully handles invalid city names."

### Conclusion (15 seconds)
```
"The project includes both a LiveKit voice agent and this browser demo.
All code is well-documented and includes error handling.
Thank you for watching!"
```

## Recording Tips

### Before Recording
1. Do a practice run
2. Know your test queries
3. Ensure good internet connection
4. Close notification apps
5. Clear browser history/autocomplete

### During Recording
1. **Speak clearly**: Enunciate for voice recognition
2. **Pace yourself**: Don't rush between actions
3. **Wait for responses**: Let voice responses complete
4. **Show, don't just tell**: Actually demonstrate features
5. **Explain briefly**: Quick explanations of what's happening

### After Recording
1. **Review the video**: Check audio and visual quality
2. **Verify length**: Should be 2-3 minutes
3. **Check demonstrations**: All features shown?
4. **Test playback**: Ensure video plays correctly

## Troubleshooting

### Video is too large
**Solution**: 
- Use MP4 format (smaller size)
- Reduce resolution to 1080p or 720p
- Use OBS with compression settings

### Voice not clear
**Solution**:
- Use external microphone
- Test audio levels before recording
- Record in quiet environment

### Recording lags
**Solution**:
- Close other applications
- Use lower recording resolution
- Ensure sufficient disk space

### Browser issues during demo
**Solution**:
- Do a practice run first
- Clear cache before recording
- Use incognito/private mode
